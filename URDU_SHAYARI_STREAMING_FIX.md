# Urdu Shayari Streaming Database Storage Fix

## Issue Identified

When streaming Urdu shayari with topic or type parameters, the generated poetry was **not being stored in the database**. The streaming endpoints were working correctly and returning poetry to the client, but no records were being saved to the MongoDB collections.

## Root Cause

The `gen_ai_function_stream()` function was only yielding streaming content to the client but **never called `save_prompt_in_db()`** after the stream completed. This is in contrast to `stream_ai_conversation()` which correctly accumulated the streamed content and saved it to the database.

### Affected Endpoints

1. **`GET /api/urdu-shayari/stream/poetry-by-topic`** - Streams poetry based on a topic
2. **`GET /api/urdu-shayari/stream/poetry-by-type`** - Streams poetry based on a type (e.g., Ghazal, Nazm)

### Affected Database Collections

- `shayari_by_topics` - Should store poetry generated by topic
- `shayari_by_types` - Should store poetry generated by type

## Changes Made

### 1. Updated `gen_ai_function_stream()` Function

**File:** `controller/urdu_ai_controller.py`

**Changes:**
- Added `all_sentences = []` list to accumulate all streamed sentences
- Modified the streaming loop to append each sentence to `all_sentences`
- After streaming completes, join all sentences and call `save_prompt_in_db()` to persist the data

**Key Addition:**
```python
# Save the complete response to database after streaming completes
if all_sentences:
    full_response = "\n".join(all_sentences)
    save_prompt_in_db(
        full_response,
        user_value,  # The topic or type that was searched
        username,
        user_time,
        "",  # character (not applicable for poetry by topic/type)
        check,  # "by_topic" or "by_type"
        "",  # character_name (not applicable)
        "",  # gender (not applicable)
    )
```

### 2. Updated `save_prompt_in_db()` Function

**File:** `controller/urdu_ai_controller.py`

**Changes:**
- Added `search_value` field to the database document for "by_topic" and "by_type" cases
- This field is required for the `get_chat_history()` function to retrieve records by topic or type

**Key Addition:**
```python
# For poetry by topic/type, store the search value for retrieval
if check in ("by_type", "by_topic"):
    json_data["search_value"] = user_prompt
```
